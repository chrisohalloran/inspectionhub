# Public demo and submission evidence protocol

Use the actual URL linked in the submission. Localhost, a deployment dashboard,
DNS records or a successful build are not public proof.

From a logged-out browser session, record:

- requested HTTPS URL, status, redirects and final HTTPS URL;
- page title and expected InspectionHub product text;
- screenshot checksum and probe-log checksum;
- explicit synthetic/demo/test label;
- named-recipient authentication;
- capability limited to the exact report version and commissioned modules;
- denial of direct private-original media;
- denial after recipient revocation;
- keyboard and screen-reader completion of the critical web journeys.

URLs written into the non-sensitive manifest must not contain report IDs,
invitations, access tokens, queries, fragments or credentials. Sensitive test
codes remain outside the manifest.

Repeat logged-out HTTPS checks for:

1. public demo video;
2. public repository and rendered README;
3. published submission description/Devpost page.

Record final URL, status and expected content for each. Rehearse the final video
and record measured duration below three minutes. Verify official Devpost fields
and rules on the challenge page immediately before submission. A drafted script
or private upload does not satisfy this gate.
