# Build Week human-validation protocol

Use synthetic/de-identified job data. Obtain appropriate participant consent
outside the public artifact set. The manifest stores only a pseudonymous
participant hash, cohort, device/browser, timing, assistance and result.

## Licensed inspector — one session

The inspector completes the representative cracked-tile plus Timber Pest/access
limitation job using only the field app after arrival. Record time, taps,
corrections, missed context, unsafe prompts, assistance and any office follow-up.
This session is also the physical-device protocol; do not double-count a
simulator replay.

## Non-expert recipients — two sessions

Starting at the authenticated report overview, ask each participant to explain
within 30 seconds:

- the major Building defect and its location;
- the separate Timber Pest result;
- the material inaccessible-area limitation;
- what is inspector opinion versus a confirmed visual observation;
- whether the report gives a buy/don't-buy recommendation (correct answer: no).

Record numerator and denominator. A prompt, correction or hidden navigation
failure remains visible. Both sessions must succeed for RC1; do not round one
success into 100%.

## Representative clients — two sessions

Each participant completes the fake/test quote-to-access journey: service and
module selection, quote, pre-inspection agreement, test payment, scheduling,
access details and named recipient. Record confusion, corrections, completion
time and whether fake/test status was understood.

## Accessibility — one moderated session or named specialist audit

Cover the complete booking/report critical path and mobile capture/review/
approval/delivery critical path using the selected assistive technology.
Record browser/device/AT versions, keyboard order, labels, state announcements,
focus, 200% reflow, reduced motion, audio-off and haptics-off behavior. Any
blocking finding keeps AC4 and the milestone blocked.
