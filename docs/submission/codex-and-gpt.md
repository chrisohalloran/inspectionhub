# How Codex and GPT-5.6 are used

## Codex as the build environment

This project is being taken from research and ideation through architecture,
implementation, test design, security review, validation and submission
packaging in Codex. The working method uses parallel, bounded reviews where the
work is independently separable, then integrates through an immutable plan and
shared gates. Adversarial agents challenge inspector ergonomics, recipient
comprehension, evidence integrity, offline recovery, security and claims before
the milestone can pass.

Codex does not replace external observation. It can write the device protocol,
test fixture and public probe, but it cannot honestly mark a physical iPhone,
human comprehension session, logged-out URL or Devpost submission complete
without observed evidence.

## GPT-5.6 inside the product

The product compares two bounded implementations using the same structured
draft schema:

1. a thin Responses API baseline; and
2. a planner-led candidate with lazy domain skills and narrow packet-read tools.

The selected runtime receives an exact, hash-bound investigation packet and may
sequence evidence review, skill loading and draft construction. It writes only
a provisional draft. Deterministic code owns schema validation, packet access,
source references, time/turn/cost budgets, staleness, persistence and redaction.
An independent verifier critiques the draft before inspector review.

Promotion is eval-driven. Development and locked-holdout cases measure critical
boundary failures, provenance, qualification fidelity, classification
authority, taxonomy integrity, verifier recall/specificity, manual fallback,
latency and cost. Any critical failure blocks promotion. The planner candidate
must also materially reduce inspector interventions without exceeding the
accepted latency/cost envelope; otherwise the thin baseline remains selected.

## Authority and privacy

The model cannot:

- create a professional classification or approval;
- access evidence outside the authorised packet;
- invite or revoke recipients;
- render, send or claim a provider outcome;
- give purchase, negotiation, valuation, repair-cost, legal or guarantee advice.

Requests use `store: false`. Evidence and transcript payloads are excluded from
telemetry/traces, and a complete inspector-authored path remains available when
the model or network is unavailable.

The live GPT-5.6 comparison is currently unproven. Deterministic tests and an
implemented harness are not a substitute for an authorised live run recorded
in the Build Week manifest.
