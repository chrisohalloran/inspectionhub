# Delivery, amendment and withdrawal exceptions

1. Identify the organization, commissioned module, approved snapshot, immutable
   report version, recipient principal and delivery package. Never operate from
   a property label or email address alone.
2. If a package is stale, incomplete or withdrawn, append the terminal event and
   prevent delivery. Do not replace its bound module set or hash.
3. A correction creates a new finding/module snapshot, approval and report
   version. The prior version remains retained and auditable.
4. A professional withdrawal is append-only and tied to the assigned eligible
   inspector with AAL2/recent-auth/device checks. Revoke recipient grants and
   cancel queued delivery for that exact version/package.
5. Reconcile provider truth before retrying an unknown send. A provider timeout
   does not establish that delivery failed.
6. Recipient UI must show the exact current delivered version or a withdrawn
   state. It must not silently redirect an old capability to a replacement
   opinion.
